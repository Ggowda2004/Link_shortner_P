Summary of changes, mistakes found, and prevention steps

1) Changes I made
- backend/app/models.py: added explicit SQLAlchemy types for `short_code` (String(32)) and `click_count` (Integer).
- backend/app/schemas.py: changed `UrlCreate.original_url` to `AnyHttpUrl` (Pydantic) for server-side URL validation.
- backend/app/services.py: cast `url.original_url` to `str` before saving to DB; `create_url()` now returns the ORM object during development.
- backend/app/routes/urls.py: `/shorten` route updated to return `{ "short_url": "http://localhost:8000/{short_code}" }` so the frontend receives `response.data.short_url`.
- frontend/src/pages/Home.tsx: added client-side URL validation (using `new URL(...)`), error messages, and improved user feedback (copy button, error UI).

2) Mistakes / root causes discovered
- Response schema mismatch: route `response_model` and actual returned value did not match (ORM object vs JSON dict). This caused runtime validation issues and broken frontend assumptions.
- `AnyHttpUrl` (Pydantic) is not a plain string; passing it directly into SQLAlchemy/psycopg2 caused `can't adapt type 'AnyHttpUrl'` on INSERT. The DB expects native types (str), not Pydantic objects.
- Some mapped columns had no explicit SQL types which can lead to ambiguous SQL generation/errors (fixed by setting `String(32)` and `Integer`).
- Minor naming typo: `limt` in `RateLimitter` (functional but confusing).
- Frontend lacked client-side validation and error display, so API failures weren't surfaced to users cleanly.

3) How to prevent these issues in future (practical rules)
- Maintain API contract: always define and document request and response schemas. Ensure a route's `response_model` matches exactly what you return.
- Convert Pydantic types to native Python values before DB usage: e.g., use `str(my_anyhttpurl)` or `model.dict()`/`model.model_dump()` when appropriate.
- Explicitly declare SQLAlchemy column types for mapped columns to avoid ambiguous DB mappings.
- Add unit tests and integration tests for key flows: shortening a URL, fetching all URLs, redirecting by short code.
- Add simple E2E tests (Playwright/Cypress) that exercise the frontend form and backend endpoints.
- Enable static analysis and linters: `mypy`/`pyright` for Python, `eslint`/`TypeScript` checks for frontend. Use pre-commit hooks.
- Add a CI workflow to run tests and linters on PRs to catch regressions early.
- Add logging and clear HTTP error messages (use consistent error shapes in responses). Surface `error.response.data.detail` on the frontend.
- Use small, focused changes. When changing input types (e.g., to `AnyHttpUrl`), audit all call sites that pass those values into DB code.

4) Quick commands to run locally
- Backend (from project root):
  python -m pip install -r backend/requirements.txt
  cd backend/app
  uvicorn main:app --reload --host 127.0.0.1 --port 8000

- Frontend:
  cd frontend
  npm install
  npm run dev
